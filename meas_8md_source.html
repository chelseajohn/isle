5 Measurements are a uniform mechanism to extract information from gauge configurations.
6 They can be called both during HMC as 'inline' measurements or afterwards as
7 'out-of-line' measurements.
8 A number of fairly general purpose measurements are defined in namespace cns.meas.
9 See section Canonical Implementation for details on how to write those measurements.
10 
11 Inline measurements represent a general hook into the Hybrid Monte Carlo function provided
12 by cns.hmc. Measurements are run after each Metropolis accept-reject step so they can be
13 used for a number of tasks. For instance, cns.hmc.hmc does not store or output intermediate
14 results by itself. Instead, a measurement can be used to save configurations to disk.
15 Other possible usage cases would be checkpointing or logging information to the terminal.
16 
17 ## Interface
18 An inline measurement must be a callable with parameters `phi, inline, itr, act, acc`, where
19 - <B>`phi`</B> - configuration,
20 - <B>`inline`</B> - `True` when called inline of HMC,
21 - <B>`itr`</B> - trajectory index,
22 - <B>`act`</B> - action given configuration `phi`,
23 - <B>`acc`</B> - `True` if trajectory was accepted, `False` otherwise.
24 - <B>`rng`</B> - A random number generator that implements cns.random.RNGWrapper.
25 
26 `phi` and `inline` are called by position whereas `itr`, `act`, `acc`, and `rng` are called
27 by name. The return value is ignored when the measurement is called in-line.
28 
29 Out-of-line measurements can have an arbitrary interface because they are only called
30 by the user. However, in order to provide a uniform way of calling both inline and
31 out-of-line measurements, the recommended implementation is
32 ```.py
33 def myMeas(phi, inline=False, **kwargs):
34  # ...
35 ```
36 for functions and
37 ```.py
38 class MyMeas:
39  # ...
40 
41  def __call__(self, phi, inline=False, **kwargs):
42  # ...
43 ```
44 for callable classes. This can accommodate the special arguments passed to inline
45 measurements which might not be available out-of-line as well as possible extra
46 arguments used with out-of-line measurements not available inline.
47 
48 ## Canonical Implementation
49 All measurements in package cns.meas follow the same implementation scheme.
50 Each measurement is a class with a name in UpperCamelCase defined in its own file of the same
51 name but in lowerCamelCase. Measurements not defined in cns.meas can have any format as long
52 as the satisfy the interface described above.
53 
54 As an example, look at the following:
55 ```.py
56 # file acceptanceRate.py
57 
58 class AcceptanceRate:
59  def __call__(self, phi, inline=False, **kwargs):
60  # ...
61 
62  # ...
63 ```
64 This allows for the class to be found by the import script and it can be used as
65 ```.py
66 # some other file
67 
68 import cns
69 import cns.meas # meas is not imported by cns base package
70 
71 accMeas = cns.meas.AcceptanceRate()
72 ```
73 If a different naming scheme is used or there are multiple measurements in one module,
74 the name of the module must be used:
75 ```.py
76 import cns
77 import cns.meas # still need this
78 
79 nonCanMeas = cns.meas.non_canonocal_module.NonCanonicalMeasurement()
80 ```
81 If a module contains extra functions or classes besides the central measurement, those can
82 only be addressed using the full module name as well.
