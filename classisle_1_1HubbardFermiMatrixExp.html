Detailed Description

Represents a fermion matrix \(\hat{M}\) for the Hubbard model with hopping matrix in an exponential.

See docs/algorithms/hubbardFermiAction.pdf for definitions. All member functions of this class are named after the corresponting matrices (sans the hat).

HubbardFermiMatrixExp needs \(\tilde{\kappa}, \phi, \tilde{\mu}, \mathrm{and}\, \sigma_\tilde{\kappa}\) as inputs and can construct the individual blocks \(\hat{K}, \hat{F}, \hat{P}, \hat{T}^{+}, \mathrm{and}\, \hat{T}^{-}\) or the full matrices \(\hat{M}, \hat{Q}\) from them.

Neither the full matrices nor any of their blocks are stored explicitly. Instead, each block is to be constructed when needed which might be expensive. The only exception is \(e^{-\tilde{\kappa}+\tilde{\mu}}\) which is cached after it has been requested through HubbardFermiMatrixExp::expKappa() for the first time.

The result of an LU-decomposition of \(\hat{Q}\) is stored in HubbardFermiMatrixExp::LU to save memory and give easier access to the components compared to a blaze::Matrix.
