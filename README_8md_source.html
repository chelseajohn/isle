3 # Short Summary
4 
5 [The Hubbard Model][hubbard] is an approximate model that describes fermions on a lattice that have a contact interaction. The Hamiltonian includes a nearest-neighbor "hopping", in which the fermions can go from one site to another, and an interaction that two fermions feel if they're on the same site. In solid-state and condensed matter physics, sites can represent atoms in some material, while the fermions are the electrons in the outer-most shells.
6 
7 It's thought that the Hubbard model approximates carbon nanostructures, such as graphene, nanotubes, and fullerenes, very well. This code is intended to allow us to study the Hubbard model, focusing on carbon nanostructures. However, it has enough generality to allow the study of the Hubbard model on arbitrary graphs. Some graphs have special features---the hexagonal lattice of graphene is [*bipartite*](https://en.wikipedia.org/wiki/Bipartite_graph), for example. In some circumstances, these special features may unlock numerical speedup.
8 
9 [hubbard]: https://doi.org/10.1098%2Frspa.1963.0204

38 ## Configuring
39 This project is installable via `pip`. However since the C++ extension is rather complex, you need to configure the build first by running
40 ```
41 python3 setup.py configure
42 ```
43 with suitable arguments. The supported arguments are (run `python3 setup.py configure --help`):
44 - *compiler*: Choose a C++ compiler. isle is tested with current versions of gcc and clang. The intel compiler is not supported because of a bug in the compiler.
45 - *build-type*: Set the CMake build type. Can be DEVEL (default), RELEASE, DEBUG
46 - *blaze*: Point to blaze installation (has to contain `blaze/Blaze.h`)
47 - *blaze-parallelism*: Select parallelism used by blaze. Allowed values are NONE (default), OMP (OpenMP), CPP (C++ threads)
48 - *blas-vendor*: Select vendor of BLAS/LAPACK library. Supported values are:
49  - `Generic` - Reference implementation (default)
50  - `Intel10_32`, `Intel10_64lp`, `Intel10_64lp_seq`, `Intel` - MKL, see https://cmake.org/cmake/help/v3.0/module/FindBLAS.html
51 - *parallel-blas*: Specify this flag if the BLAS/LAPACK implementation is parallelized. Otherwise blaze might parallelize on top of it which can slow down the program.
52 
53 This does not actually run CMake to configure the C++ build but merely performs some rudimentary checks and saves the parameters.
54 
55 Note that specifying the compiler or build type when running `python3 setup.py build` does not work; they need to be set when configuring.
